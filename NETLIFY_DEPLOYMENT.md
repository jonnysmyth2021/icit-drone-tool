# Netlify Deployment

This application is a Next.js 16 App Router project deployed from GitHub using
Netlify Continuous Deployment. Netlify's automatic OpenNext adapter provides
SSR, Server Actions, App Router route handlers, middleware/proxy execution, and
image optimization. Do not install or pin `@netlify/plugin-nextjs`.

## Build configuration

| Setting | Value |
| --- | --- |
| Build command | `pnpm run build` |
| Publish directory | `.next` |
| Node.js | `22` |
| Package manager | `pnpm@10.28.0` |

These values are committed in `netlify.toml` and `package.json`. The `.next`
publish directory is the documented value for a hybrid/SSR Next.js site on
Netlify; OpenNext transforms it into the required functions and deploy output.

## Required environment variables

Add variables in **Project configuration → Environment variables**. Use **All
scopes** so values are available during builds, in functions, and at runtime.
Apply them to Production and any Deploy Preview contexts that need to work.

### Public configuration

These values are embedded in the browser bundle and must not be marked as
secrets:

| Variable | Required | Purpose |
| --- | --- | --- |
| `NEXT_PUBLIC_SUPABASE_URL` | Yes | Supabase project URL. |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | Yes* | Current Supabase browser/server publishable key. |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Alternative* | Legacy key; use only when no publishable key is available. |

*Set either `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` or
`NEXT_PUBLIC_SUPABASE_ANON_KEY`. The publishable key is preferred.

### Secret server configuration

Import these with **Contains secret values** enabled:

| Variable | Required | Purpose |
| --- | --- | --- |
| `OPENAI_API_KEY` | Yes | Server-side intelligence assessment. |
| `OPENSKY_CLIENT_ID` | Yes | OpenSky OAuth client credentials. |
| `OPENSKY_CLIENT_SECRET` | Yes | OpenSky OAuth client credentials. |
| `OPENSKY_TIMEOUT_MS` | No | OpenSky request timeout; defaults to 8000 ms. |
| `AIRPLANES_LIVE_TIMEOUT_MS` | No | Airplanes.live request timeout; defaults to 8000 ms. |
| `OPENWEATHER_API_KEY` | No | Enables the current weather and wind snapshot stored with generated reports. |
| `SUPABASE_SERVICE_ROLE_KEY` | No | Correctly named server-only Supabase administrative key. |
| `OPENAI_INTELLIGENCE_MODEL` | No | Overrides the default `gpt-5.4-mini`. |
| `OPENAI_VISUAL_INTELLIGENCE_MODEL` | No | Overrides the multimodal model used for post-upload photo analysis; otherwise uses `OPENAI_INTELLIGENCE_MODEL`. |
| `DATABASE_URL` | No | Reserved for the currently unused Drizzle/Postgres module. |

Never add secrets to `netlify.toml` or commit a populated `.env` file. Copy-ready
key templates are available in `.env.netlify.public.example` and
`.env.netlify.secrets.example`.

## Deployment steps

1. Push the repository to GitHub.
2. In Netlify, choose **Add new project → Import an existing project → GitHub**.
3. Select the repository and production branch.
4. Confirm framework detection reports **Next.js**.
5. Confirm build command `pnpm run build` and publish directory `.next`.
6. Import the public and secret environment-variable batches separately.
7. Trigger **Deploy site**. If variables were added after a build, redeploy; a
   previous artifact does not acquire newly added `NEXT_PUBLIC_*` values.
8. Verify login, `/report`, `/review`, `/api/aircraft`, and
   `/api/intelligence` in the deployed site.

## Supabase prerequisites

The message `Supabase is not configured yet` is generated by the application,
not Netlify. It appears when `NEXT_PUBLIC_SUPABASE_URL` is missing or neither
supported Supabase public key is present.

The target Supabase project must also have:

- the migrations in `supabase/migrations` applied;
- Auth enabled and the required users created;
- the `user_profiles` roles expected by the application;
- a `report-media` Storage bucket with suitable policies;
- the Netlify production URL added to Supabase Auth redirect/site URLs where
  applicable.

## Runtime compatibility

- `proxy.ts` is the Next.js 16 network-boundary proxy. It uses standard
  `NextRequest`/`NextResponse` and Supabase SSR APIs and is portable to
  Netlify's OpenNext runtime.
- `/api/aircraft` and `/api/intelligence` are standard App Router Node.js route
  handlers and become Netlify Functions automatically.
- Server Actions under `app/actions` use standard Next.js and Supabase APIs.
- There are no Edge Runtime declarations, Vercel Blob, Vercel KV, Edge Config,
  `@vercel/functions`, or other Vercel runtime APIs.
- There are currently no custom redirects, rewrites, or headers. Add portable
  application rules through `next.config.mjs`, or Netlify-specific static rules
  through `netlify.toml`; do not restore `vercel.json`.
- Next.js image optimization is enabled. Netlify will back `next/image` with
  Netlify Image CDN. Evidence previews use native `img` elements because their
  sources are local data URLs or short-lived signed URLs.

## Known issues

- OpenSky connectivity can vary by cloud runtime. The aircraft service falls
  back automatically to Airplanes.live and reports both provider statuses.
- Airplanes.live's public API has usage limits and no uptime SLA. Successful
  aircraft responses are cached briefly to reduce duplicate requests.
- OpenSky, Airplanes.live, and OpenAI are external services with independent
  rate limits and availability.
- `next/font/google` downloads Geist during the build. A build environment that
  cannot reach Google Fonts will fail, even though Netlify normally permits the
  request.

## Troubleshooting

### “Supabase is not configured yet”

1. Confirm `NEXT_PUBLIC_SUPABASE_URL` exists in the affected deploy context.
2. Confirm either `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` or
   `NEXT_PUBLIC_SUPABASE_ANON_KEY` exists.
3. Confirm the variables include the Builds scope or use All scopes.
4. Redeploy after changing `NEXT_PUBLIC_*` values.
5. Inspect the deploy log to ensure the build used the expected context.

### Login fails after configuration

Check the Supabase URL/key pairing, Auth users, `user_profiles`, row-level
security policies, and permitted site/redirect URLs. A key from a different
Supabase project will not work with the configured URL.

### API route returns 404

Confirm Netlify detected Next.js and installed its automatic OpenNext adapter.
The publish directory must be `.next`; do not deploy this application as a
static export and do not set `NETLIFY_NEXT_PLUGIN_SKIP=true`.

### Aircraft are unavailable

Inspect Netlify Function logs for structured `aircraft` provider events. Verify:

- `provider`, `status`, `httpStatus`, `durationMs`, and `timeout`;
- whether `fallback_triggered` was logged;
- `diagnostics.openskyStatus` and `diagnostics.airplanesStatus` in the API response;
- the selected provider and parsed aircraft count.

If DNS succeeds but `UND_ERR_CONNECT_TIMEOUT` appears again, provide Netlify
and OpenSky support with the destination, function region, timestamp, and
nested Undici error.

### Build fails

1. Check Node 22 and pnpm 10.28.0 are selected.
2. Confirm all public variables are available to the build context.
3. Clear the Netlify dependency/build cache and retry.
4. Check for Google Fonts download failures.
5. Run `pnpm install --frozen-lockfile`, `pnpm exec tsc --noEmit`, and
   `pnpm run build` locally with Node 22.

### Middleware/session problems

Check Netlify function/edge logs and browser cookies. The proxy matcher excludes
Next.js assets and image files but intentionally runs for pages and API routes
to refresh Supabase sessions.
